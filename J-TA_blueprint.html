01 · Executive Summary

Tổng quan dự án J - TA

Mục tiêu là tạo một nền tảng booking/private charter theo kiến trúc hiện đại, có website public, dashboard admin, API backend, tài liệu Swagger và database có khả năng mở rộng.

Nguyên tắc quan trọng: chỉ tái dựng bố cục, UX pattern, luồng chức năng và kiến trúc hệ thống. Không sao chép logo, hình ảnh, nội dung, thương hiệu, icon độc quyền hoặc dữ liệu thật của website mẫu.

02 · System Architecture

Kiến trúc tổng thể cần có

Kiến trúc tách thành 3 lớp: frontend public, backend API, dashboard quản trị. Dữ liệu dùng PostgreSQL, Prisma migration, Swagger làm hợp đồng API.

Frontend · Next.js

App Router, SSR/SSG, route đa ngôn ngữ, component UI tách riêng, content layer để thay nội dung không phá layout.

apps/web ├─ app/[locale] ├─ components ├─ content/mock ├─ lib/api ├─ hooks └─ styles

Core Platform · API + DB + Docs

NestJS xử lý business logic, Prisma quản lý schema, PostgreSQL lưu dữ liệu, Swagger xuất tài liệu API. Redis dùng cache/session/rate limit.

Client UI ↓ Next.js Web / Admin ↓ NestJS REST API ↓ Prisma ORM ↓ PostgreSQL + Redis + Object Storage ↓ Swagger / OpenAPI / Audit Logs

Admin Dashboard

Dashboard custom để quản trị toàn hệ thống: quote, booking, aircraft, partner, content, media, legal, setting.

apps/admin ├─ dashboard ├─ quotes ├─ bookings ├─ content ├─ partners ├─ media └─ settings
